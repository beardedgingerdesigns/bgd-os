# Milestone v1.0: AIOS UI v0→v2 — Local Operator Command Center

**Status:** ✅ SHIPPED 2026-05-22
**Phases:** 1–4
**Total Plans:** 9 (Phase 4); Phases 1–3 shipped pre-`.planning/` setup without per-plan artifacts

## Overview

Built Justin's personal AIOS UI as a four-slice progression: read-only filesystem render → live sync + daily triage → operator capture + ritual launcher → bidirectional hub with brief-hydrated chat and staged ingestion. The UI runs locally on `localhost:3000`, reads `clients.yaml` and a constellation of markdown sources as filesystem-source-of-truth, and shells out to Claude Code skills for every AI operation. Final shape: an operator workspace where Glance (MRR + activity), Drill (per-project pages), Act (chat + capture + triage) all live without a context switch.

Phase 4 (Bidirectional Hub) was the headline build — added pre-built indexed briefs hydrating per-project chat, staged ingestion via `{wiki}/raw/aios/`, a Receipt feed as the trust mechanism, per-project Communications surface with triage overrides, and a Pending Ingestion bridge to the `llm-wiki` curator.

## Milestone Summary

**Decimal Phases:** None inserted.

**Key Decisions** (ADRs 0001-0005, locked):

- ADR 0001 — AIOS UI architecture: local-only Next.js, additive viewer not replacement, CLI subprocess for chat, in-memory index + chokidar, drill-down nav, no DB.
- ADR 0002 — MRR data model: structured `mrr_monthly` field, `paying + active` filter rule, dealers-as-Projects.
- ADR 0003 — Bidirectional store model: UI is the seam between operator inputs and durable storage; two-way pipe.
- ADR 0004 — Staged ingestion via `raw/aios/`: AIOS writes only to staging; `llm-wiki` ingest pass owns promotion to curated structure.
- ADR 0005 — Chat hydration via pre-built indexed briefs: supersedes ADR 0001 §6 auto-load-on-expand mechanism; lazy-build now, watcher removed during gap-closure.

**Issues Resolved during milestone:**

- GAP-04-01 (Communications parser keyed to obsolete `Thread:` markers) — fixed by parsing `TODOS_JSON_START/END` envelope as primary data source.
- GAP-04-02 (Brief watcher fan-out + universal `/load-project` exit 1 + SQLite lock contention) — fixed by ripping the watcher entirely; lazy build + Refresh button replaces eager rebuild.
- Build error from client-bundle leak (`child_process` imported via daily-ingest.ts into client triage-output.tsx) — fixed by splitting pure parser into `lib/skills/todos-envelope.ts`.
- 2 Critical + 9 Warning findings from `/gsd-code-review 4` — all fixed atomically (CR-01 stderr drain, CR-02 slug validation, WR-01..09 various).

**Issues Deferred:**

- 6 Info findings from code review (extractTextDelta dedup, receipts.jsonl rotation policy, slugify non-Latin handling, Math.random() id collisions, relativeTime helper dedup, ProjectReceiptsSlice cap edge case) — backlogged for a future polish pass.
- Phases 1–3 retroactively documented in ROADMAP without per-plan SUMMARY files — they predate `.planning/` setup. Acceptable historical convention.

**Technical Debt Incurred:**

- WR-01 fix landed atomic write + corruption-tolerant read for `sessions.json`, but multi-tab lost-update race (concurrent writes from two browser tabs) deliberately NOT addressed — would require process-wide mutex; out of scope for single-operator AIOS.
- Brief watcher code (`lib/indexer/watcher.ts`) deleted in gap-closure; `lib/indexer/build-brief.ts` retained for the lazy path. If eager rebuild is wanted again, a from-scratch design is needed (the per-project diff + rate-limit work was never built).
