    Deferred dispatch retry logic (runs at the START of each triage pass, before new thread processing):
    1. For each entry in `deferredDispatches`:
       a. Re-read clients.yaml for the slug to get current bucket value

       c. If bucket is still `prospects`: check prospect doc status. If `active`/`warm`/`cold`: retry prospect dispatch normally. If `converting`: keep deferred (conversion still in progress). If `converted`/`dead`: clear deferred entry (terminal, intel no longer relevant to this surface).

    - Write the updated prospect doc back to disk
