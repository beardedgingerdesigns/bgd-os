<objective>
Create the session metrics accumulator hook and supporting infrastructure for the end-of-session state system (STATE-03).

Purpose: The metrics hook runs after every substantive tool call and tracks session activity (edits, writes, commits, user messages) in a temp file. The SessionEnd hook (Plan 02) reads these metrics to decide whether the session was substantive enough to warrant a STATE.md write. This plan also creates the `state/` directory where synced STATE.md copies will land, and the LLM prompt template used for summarization.

Output: PostToolUse hook script at `~/.claude/hooks/state-session-metrics.js`, state prompt template at `scripts/state-hook/state-prompt.md`, empty `state/` directory with `.gitkeep`.
</objective>

  <files>~/.claude/hooks/state-session-metrics.js</files>
  <read_first>
    - ~/.claude/hooks/gsd-context-monitor.js (reference implementation for PostToolUse hook pattern: stdin parsing, session_id sanitization, temp file I/O, timeout guard)
    - .planning/phases/05-end-of-session-state-hook/05-RESEARCH.md (Pattern 1: PostToolUse Metrics Accumulator, Substance Threshold Logic)
  </read_first>
  <action>
    Create `~/.claude/hooks/state-session-metrics.js` following the exact structural pattern of `gsd-context-monitor.js`:

    1. Shebang: `#!/usr/bin/env node`
    2. Version comment: `// gsd-hook-version: 1.0.0`
    3. Description comment block explaining: this PostToolUse hook accumulates session metrics for the end-of-session STATE.md generator. It runs after Edit, Write, MultiEdit, and Bash tool calls. Metrics are stored in `/tmp/claude-state-{session_id}.json`.

    4. Require `fs`, `os`, `path` (matching gsd-context-monitor.js pattern).

    5. stdin timeout guard: 5 seconds (matching research recommendation; gsd-context-monitor uses 10s but this hook is simpler).

    6. Read stdin JSON. Extract `session_id` and `cwd`. Sanitize `session_id` with the same path-traversal check from gsd-context-monitor (`/[/\\]|\.\./.test(sessionId)`). Exit 0 on invalid.

    7. Compute metrics file path: `path.join(os.tmpdir(), 'claude-state-' + sessionId + '.json')`.

    8. Read existing metrics file or initialize default: `{ edits: 0, writes: 0, bash: 0, commits: 0, user_messages: 0, cwd: data.cwd || '' }`. When reading existing, always update `cwd` to the latest value from stdin.

    9. Increment counters based on `data.tool_name`:
       - `'Edit'` or `'MultiEdit'` -> `metrics.edits++`
       - `'Write'` -> `metrics.writes++`
       - `'Bash'` -> `metrics.bash++`, and if `data.tool_input?.command` includes the string `'git commit'`, also `metrics.commits++`

    10. Write metrics JSON back with `fs.writeFileSync`.

    11. Exit 0.

    The entire script must be synchronous (no async/await, no promises). All errors caught silently with exit 0 (hooks must never block Claude Code).
  </action>
  <verify>
