    Create `~/.claude/hooks/state-session-end.js` as a Node.js script. This is the most complex hook in the system. Follow the structural patterns from gsd-context-monitor.js for stdin parsing, but this hook has unique requirements.

    HIGH-LEVEL FLOW:
    1. Read stdin JSON (with 10s timeout guard -- this hook does more work)
    2. Extract session_id, cwd, transcript_path from stdin
    3. Sanitize session_id (same path-traversal check as metrics hook)
    4. Read metrics from `/tmp/claude-state-{session_id}.json`
    5. Apply substance threshold: `(edits + writes) >= 1 OR commits >= 1 OR user_messages >= 3`
    6. If below threshold, clean up temp file and exit 0
    7. Resolve cwd to project via clients.yaml reverse lookup
    8. If no match or cwd is claude-os root, clean up and exit 0
    9. Find wiki root by walking docs_paths for wiki markers
    10. Read transcript_path JSONL for context (last 200 lines max, streaming)
    11. Build LLM prompt with transcript excerpt + system prompt from state-prompt.md
    12. Invoke `claude -p --bare --model haiku` with the prompt via execSync
    13. Hash-compare generated content with existing STATE.md at wiki root
    14. If different (or no existing file), write to wiki root and state/<slug>.md
    15. Clean up temp metrics file
    16. Exit 0

    DETAILED IMPLEMENTATION NOTES:

    **stdin parsing (step 1-3):** Use the same `process.stdin.on('data'/'end')` pattern with a `setTimeout` guard (10 seconds, not 5 -- this hook does real work). Parse JSON, extract fields. Exit 0 on any parse error.

    **Metrics read (step 4-6):** Read `/tmp/claude-state-{session_id}.json` with try/catch. If file doesn't exist (session had no tool calls), treat as below threshold. The substance check function `isSubstantive(m)` returns true when `(m.edits + m.writes) >= 1 || m.commits >= 1 || m.user_messages >= 3`.

    **User messages count (step 4 addendum):** The PostToolUse hook cannot count user messages (it only fires on tool use). To get the user message count, count lines in the transcript JSONL that parse as JSON with `"type":"user"`. Do this BEFORE the substance check. Read the transcript at `transcript_path` line by line, count entries where `JSON.parse(line).type === 'user'`. Update `metrics.user_messages` with this count. Use `fs.readFileSync` and split on newlines -- transcripts at session end are complete and unlikely to be hundreds of MB for typical sessions. If the file is large (> 5MB), read only the first 5MB.

    **CWD-to-project lookup (step 7-8):**
    - Define `CLAUDE_OS_ROOT` as the directory containing `clients.yaml`. Resolve it by walking up from a known path. Hardcode `/Users/justinlobaito/repos/claude-os` (this is a user-level hook for one operator, not a distributable package).
    - If `cwd === CLAUDE_OS_ROOT` or cwd starts with `CLAUDE_OS_ROOT + '/.planning'`, exit 0 (skip claude-os sessions per research Pitfall 3).
    - Read `clients.yaml` using `fs.readFileSync`. Parse YAML. Since `js-yaml` is not available as a global module, implement a minimal YAML parser function that handles the `clients.yaml` structure: top-level `clients:` array of objects with `slug`, `name`, `projects` (array of objects with `slug`, `name`, `docs_paths` array of strings). The parser needs to handle indentation-based nesting, string values (quoted and unquoted), and arrays with `- ` prefixed items. Alternatively, require js-yaml from the aios-ui node_modules path: `require('/Users/justinlobaito/repos/claude-os/aios-ui/node_modules/js-yaml')`.
    - Iterate all clients and projects. For each project, resolve each `docs_path` entry: if it starts with `/`, use as-is; otherwise, join with `CLAUDE_OS_ROOT`. Check if the resolved docs_path starts with `cwd + '/'` OR if `cwd` starts with the resolved docs_path. Also check if `cwd` matches the repo root implied by a docs_path (e.g., docs_path `/Users/.../repos/mr-gym/docs` implies repo root `/Users/.../repos/mr-gym` -- check if cwd equals or is under that repo root by stripping the last path segment if it is `docs`, `wiki`, or `docs/wiki`).
    - Return `{ clientSlug, projectSlug, projectName, docsPath }` for the first match. If no match, exit 0.

    **Wiki detection (step 9):**
    - Given the matched `docsPath`, check if it contains a `wiki/` subdirectory with `WIKI-CLAUDE.md`, or if `WIKI-CLAUDE.md` exists directly in the docs path. The wiki root is the directory containing (or whose `wiki/` child contains) `WIKI-CLAUDE.md`. If no wiki markers found, the `docsPath` itself is the target directory for STATE.md.
    - Define `wikiRoot` as the detected wiki root, or `docsPath` as fallback.

    **Transcript reading (step 10):**
    - Read the file at `transcript_path` with `fs.readFileSync` (up to 5MB). Split on newlines. For each line, try `JSON.parse`. Collect the last 50 entries of type `"user"` or `"assistant"` (skip `queue-operation`, `attachment`, `file-history-snapshot`, `last-prompt`, `ai-title`). For assistant entries, extract only `message.content` items of type `"text"` (skip tool_use blocks). Truncate each text block to 2000 chars. Join all collected entries as a transcript excerpt string with `[User]:` and `[Assistant]:` prefixes.

    **LLM invocation (step 11-12):**
    - Read the system prompt from `CLAUDE_OS_ROOT + '/scripts/state-hook/state-prompt.md'`.
    - Build the full prompt as: system prompt + newline + `\n---\n\nProject: {projectName}\nDate: {ISO date today}\n\nTranscript excerpt:\n\n` + transcript excerpt.
    - Invoke `claude -p --bare --model haiku` using `child_process.execSync`. Pass the prompt via stdin using `execSync('claude -p --bare --model haiku', { input: fullPrompt, encoding: 'utf8', timeout: 25000 })`. The 25s subprocess timeout gives buffer within the 30s hook timeout.
    - Capture stdout as the generated STATE.md content. Trim whitespace.

    **Idempotency check (step 13):**
    - Compute SHA-256 hash of the generated content using `require('crypto').createHash('sha256').update(content).digest('hex')`.
    - Read existing STATE.md at `wikiRoot + '/STATE.md'` if it exists. Hash it. If hashes match, skip write, clean up, exit 0.

    **Dual write (step 14):**
    - Write to `wikiRoot + '/STATE.md'` using atomic temp+rename: write to `wikiRoot + '/STATE.md.tmp'`, then `fs.renameSync` to `wikiRoot + '/STATE.md'`.
    - Write to `CLAUDE_OS_ROOT + '/state/' + projectSlug + '.md'` using the same atomic pattern.
    - Create `state/` directory if it doesn't exist (`fs.mkdirSync` with `{ recursive: true }`).

    **Cleanup (step 15):**
    - Delete `/tmp/claude-state-{session_id}.json` with try/catch (ignore errors).

    **Error handling:** Wrap the entire main logic in try/catch. On any error, write a one-line diagnostic to stderr (`process.stderr.write`) and exit 0. Hooks must NEVER exit non-zero or throw uncaught exceptions.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| stdin -> hook script | session_id and cwd from Claude Code stdin -- untrusted input |
| hook script -> filesystem | STATE.md writes to wiki root and state/ -- must validate paths |
| hook script -> claude CLI | LLM summarization subprocess -- must use --bare to prevent recursive hooks |
| transcript JSONL -> LLM prompt | Transcript content passes through to LLM -- read-only, no injection risk since LLM is generating markdown |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | Tampering | session_id path traversal | mitigate | Reject session_id containing `/`, `\`, or `..` before constructing any file path |
| T-05-02 | Tampering | cwd-based file write | mitigate | Validate cwd against known repo paths from clients.yaml; never write outside wiki roots or state/ |
| T-05-03 | Tampering | recursive hook firing | mitigate | Use `claude -p --bare` which skips all hooks, preventing infinite recursion |
| T-05-04 | Denial of Service | SessionEnd timeout | mitigate | 30s hook timeout + 25s subprocess timeout; graceful exit on timeout |
| T-05-05 | Denial of Service | PostToolUse latency | mitigate | 5s timeout, synchronous I/O only, no network calls |
| T-05-06 | Information Disclosure | STATE.md content | accept | STATE.md contains project status summaries from the user's own sessions -- no PII beyond what's already in the transcript |
| T-05-SC | Tampering | npm/pip/cargo installs | accept | No new packages installed -- all dependencies are Node.js built-ins or already-installed js-yaml from aios-ui |
</threat_model>
