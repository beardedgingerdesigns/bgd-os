<objective>
Build the flag resolution UI that surfaces contradictions to the operator for resolution rather than auto-overwriting wiki content (WIKI-04). Flagged files appear inline in the existing WikiIngestModal with side-by-side comparison and resolution action buttons. A lightweight API endpoint handles "Accept incoming" (targeted promote) and "Keep existing" (mark resolved) actions.

Purpose: Without operator resolution, contradictions would either be silently swallowed (skip) or auto-overwrite (promote), both violating WIKI-04's requirement that contradictions surface to the human. This completes the two-layer intelligence pipeline.

Output: WikiFlagDetail component, enhanced WikiIngestModal, resolve API endpoint, and component tests.
</objective>

    Create a new API route at aios-ui/app/api/wiki/ingest/[client]/[project]/resolve/route.ts. This handles operator resolution of flagged contradictions.

    Route exports: dynamic = "force-dynamic", runtime = "nodejs", and async POST handler.

    POST handler accepts params Promise of { client: string; project: string }. Body shape: { file: string; action: "accept" | "skip" }.

    Validation: file must be a non-empty string. action must be "accept" or "skip". Return 400 JSON error for invalid input.

    Resolve wikiPath via resolveProjectWikiPath(client, project) from @/lib/data/wiki. Return 400 if no wiki path.

    Action "skip": The flagged file stays in raw/aios/ (immutable per ADR 0004 -- we do NOT delete it). To prevent it from being re-flagged on the next ingest run, write a resolution record to a new file at {wikiPath}/raw/aios/.resolved/{filename}.json with content { action: "skip", resolvedAt: ISO timestamp, reason: "operator chose to keep existing wiki content" }. Create the .resolved/ directory if it does not exist. Return { ok: true, action: "skip", file }.

    Action "accept": The operator wants to promote the flagged content. Spawn a targeted ingest for just this one file. Use the existing claude subprocess pattern from wiki-ingest.ts (spawn claude with --print, --permission-mode bypassPermissions). The prompt should be: "/ingest-aios-drops {wikiPath} --file {filename}" where filename is the specific raw/aios/ file to promote. However, the current skill does not support --file targeting. ALTERNATIVE APPROACH: Instead of subprocess, directly invoke the /llm-wiki skill on the specific file. Simpler: spawn claude with prompt "/llm-wiki {wikiPath}/raw/aios/{filename}" to promote just that file. After successful promotion, write a resolution record to {wikiPath}/raw/aios/.resolved/{filename}.json with { action: "accept", resolvedAt: ISO timestamp }. Return { ok: true, action: "accept", file }.

    Import resolveProjectWikiPath from @/lib/data/wiki. Import spawn from child_process. Import fs from fs/promises and path.

    The .resolved/ directory acts as the resolution ledger. The next ingest run can check for .resolved/{filename}.json to skip already-resolved files. Update the SKILL.md to document this: in Step 3.1 (read file contents), add "If {wikiPath}/raw/aios/.resolved/{filename}.json exists, skip this file (already resolved by operator)."

    NOTE on ADR 0004 immutability: We do NOT modify or delete files in raw/aios/. The .resolved/ subdirectory is a separate metadata layer, not a mutation of the raw drops themselves. This respects the immutability contract.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser -> resolve API | Operator clicks resolution button; API receives file name and action |
| Resolve API -> filesystem | API writes .resolved/ records and spawns claude subprocess |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-07 | Tampering | resolve/route.ts file param | mitigate | Validate file param is a simple filename (no path separators, no ..); reject if file contains / or \. Use path.basename() as defense-in-depth before constructing .resolved/ path. |
| T-07-08 | Spoofing | resolve/route.ts | accept | Single-operator, local-only system. No auth needed per ADR 0001 S1. |
| T-07-09 | Denial of Service | /llm-wiki subprocess on accept | accept | Operator explicitly chooses to promote; subprocess cost is expected. Same pattern as existing ingest. |
| T-07-10 | Tampering | .resolved/ directory | accept | .resolved/ is a metadata layer, not a security boundary. Operator could manually delete .resolved/ files to re-trigger evaluation. This is acceptable behavior. |
</threat_model>
