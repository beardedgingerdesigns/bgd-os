<objective>
Make Capture wiki-aware (HUB-05) and emit a receipt for every successful capture (HUB-06). When the Project has a wiki in its docs_paths, the UI writes directly to `{wiki}/raw/aios/capture-YYYY-MM-DD-<slug>.md` via the lib/raw-drops.ts helper from 04-01. When no wiki, fall back to the existing `/capture` skill subprocess. Both branches honor the HUB-06 contract: a receipt is only emitted when an absolute file path is known. The subprocess branch tries multiple regex patterns; if none match, the capture succeeds but no receipt is emitted (preferable to a receipt with an empty path).

Purpose: Today, Capture spawns `/capture` and the result "vanishes" (operator's complaint). After this plan: every capture that has a parseable destination is durable in a place the operator can see, the receipt feed has a path link, and the brief indexer (04-06) will pick the new file up on next rebuild.

Output: Wiki-aware capture path + receipt emission + resolveProjectWikiPath helper + updated tests.
</objective>

       ```typescript
       export async function resolveProjectWikiPath(
         clientSlug: string,
         projectSlug: string,
       ): Promise<string | null> {
         const project = await getProject(clientSlug, projectSlug)
         if (!project) return null
         const docsPaths = resolveDocsPaths(project.docs_paths)
         for (const dp of docsPaths) {
           const wiki = await detectWiki(dp)
           if (wiki) return wiki.rootPath
         }
         return null
       }
       ```
