<objective>
Build the background indexer (ADR 0005): a per-project brief builder that invokes `/load-project` as a subprocess and writes `.aios-cache/briefs/<slug>.md`, plus a chokidar watcher that rebuilds affected briefs on changes to `clients.yaml`, `references/`, `memory/`, each project's `docs_paths`, and each project's `wiki/raw/aios/` directory with 500ms debounce. Bootstrap both via `instrumentation.ts`.

Purpose: HUB-02. The chat panel in 04-07 reads the cached brief on every bootstrap. Without this indexer, the cache is empty and chat falls back to the slower in-process build (which is what `chat-brief.ts` does today). Pre-built briefs make Project chat instant.

Output: Two new lib/indexer files, an extended instrumentation.ts, unit tests for both, and a graceful fallback to the existing buildProjectBrief when `/load-project` subprocess fails.
</objective>

<interfaces>
lib/indexer/build-brief.ts exports:
- briefPathFor(clientSlug: string, projectSlug: string): string
  Returns path.join(AIOS_CACHE_DIR or process.cwd()+'.aios-cache', 'briefs', `${clientSlug}__${projectSlug}.md`).
- buildBriefFor(clientSlug: string, projectSlug: string, opts?: { claudeBin?: string; timeoutMs?: number }): Promise<{
    status: 'success' | 'failed' | 'timeout' | 'fallback';
    filePath: string;
    builtAt: string;
    durationMs: number;
    error?: string;
  }>
  Primary path: spawn 'claude --print --output-format stream-json --include-partial-messages --verbose "/load-project <projectSlug>"' with 60s timeout. Aggregate text deltas. On success, write to briefPathFor(...).
  Fallback path: on subprocess failure/timeout, call buildProjectBrief(clientSlug, projectSlug) from lib/skills/chat-brief.ts and write THAT instead. status='fallback' so callers can log.
  Returns the file's absolute path either way (file is always written even on fallback).

lib/indexer/watcher.ts exports:
- startBriefWatcher(): Promise<{ close: () => Promise<void> }>
  Idempotent singleton (mirror lib/watcher.ts pattern with module-scope `activeBriefWatcher`).
  Watches: CLIENTS_YAML_PATH, REFERENCES_DIR, MEMORY_ROOT, and for each project: each resolved docs_path + each wiki's `raw/aios/` dir.
  On change → debounce 500ms keyed by (clientSlug, projectSlug) → buildBriefFor(...).
- stopBriefWatcher(): Promise<void>

instrumentation.ts changes:
- After existing startWatcher() call, dynamic-import startBriefWatcher and call it.
- Both are best-effort; failures logged but do not crash boot.
</interfaces>
