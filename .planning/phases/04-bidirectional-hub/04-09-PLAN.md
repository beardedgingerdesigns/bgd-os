<interfaces>
lib/skills/wiki-ingest.ts exports:
- runWikiIngest({ wikiPath, claudeBin?, timeoutMs?, onStdout? }): Promise<WikiIngestResult>
  Default timeoutMs = 300_000 (5 min, mirroring rituals).
  Spawns claude with args: --print, --permission-mode bypassPermissions, --output-format stream-json, --include-partial-messages, --verbose, then the prompt string "/ingest-aios-drops {wikiPath}".
  WikiIngestResult shape: { status: 'success' | 'failed' | 'timeout'; output: string; exitCode: number; durationMs: number; error?: string; summary?: { promoted: string[]; deferred: string[]; contested: string[] } }
  Summary is parsed from the aggregated output via the INGEST_SUMMARY_START / INGEST_SUMMARY_END marker pattern (analogous to TODOS_JSON_START in daily-ingest.ts).

POST /api/wiki/ingest/[client]/[project]:
- Body: none
- Resolves wikiPath via resolveProjectWikiPath; if null returns 400 { error: 'no wiki path resolved for this project' }
- Streams SSE: start event, chunk events (text from subprocess), done event with the WikiIngestResult JSON
- On result.status === 'success': appendReceipt with kind=wiki_ingest, project_slug, file_written=wikiPath+'/log.md' (canonical target of ingestion), excerpt=one-line summary; then invalidationBus.publish project-scoped

PendingIngestionSection (server component) props:
- { clientSlug: string; projectSlug: string; wikiPath: string | null }

WikiIngestModal (client component) props:
- { clientSlug: string; projectSlug: string; open: boolean; onClose: () => void }

RunIngestButton (client component) props:
- { clientSlug: string; projectSlug: string; count: number }

ProjectReceiptsSlice (server component) props:
- { projectSlug: string }
