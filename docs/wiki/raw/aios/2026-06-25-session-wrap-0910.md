## Decisions

- Snooze applies to persistent todos only — ephemeral todos regenerate daily so snooze would be meaningless
- "Accept" label stays on ephemeral cards, "Do It" on persistent — different semantics (accepting a triage suggestion vs. executing a task)
- Do It (generic) deferred per ADR — wait for patterns from email/calendar
- Action type detected via keyword scan on summary (send/email/reply → email; schedule/meeting/call → calendar)
- Primary action is a visible button (Do It or Done), secondary actions in kebab menu — matches triage card pattern
- `pending-todo-actions.ts` duplicates subprocess spawn logic from `todo-actions.ts` rather than refactoring a shared runner — keeps the two systems independent
