## Decisions

- Use system clock (`toLocaleDateString` with weekday) for date injection instead of YYYY-MM-DD string — prevents day-of-week guessing errors.
- Session expiry = midnight boundary, not hour-based TTL. Keeps conversational context alive during a workday but guarantees fresh context each morning.
- Kept `--resume` model within same day (vs. ripping it out entirely for standalone calls) — Justin values conversational continuity during a work session.
- CLAUDE.md staged ingestion rule updated: when working inside claude-os itself, write directly to curated wiki pages. Staging rule only applies to remote writes into other project wikis.

## Struggles

- Initially used `toLocaleDateString('en-CA')` (YYYY-MM-DD only) for date injection. Model guessed "Saturday" for a Sunday. Had to switch to full day-name format. Root cause: models can't reliably compute day-of-week from a date string.
- Deleted raw/ files during wiki ingest — violated the "raw/ is immutable" rule. Had to restore from git. The pattern treats raw/ as a permanent source archive, not a staging inbox.
