# Overwhelmed by AI? The Lean Tech Stack and Anti-Overwhelm Mindset

**URL:** YouTube (@nateherk) -- "Overwhelmed by AI? Just Copy My Tech Stack"
**Published:** 2026 (recent)
**Type:** YouTube video + community post

## Key Frameworks and Patterns

### The Tech Stack Hierarchy
- **S-Tier (daily drivers):** Claude Code is the top daily driver and effectively acts as the operating system. Glido (GLO) for speech-to-text.
- **A-Tier (weekly use):** Codex, Claude, Hermes Agent, Perplexity, Grok. Complement the Claude Code setup.
- **B-Tier (specialists):** OpenRouter, ElevenLabs, HeyGen, GPT-Image for specific tasks (avatars, voices, research).
- **Graduated (no longer used):** ChatGPT, OpenClaw -- tools that were replaced by the stack above.

### The Anti-Overwhelm Framework
Every person following AI right now is overwhelmed. The distinction that matters:

- **"What" vs "How."** Most new tools only need the "what" -- you see the title, understand what it does, move on. The "how" is reserved for stuff that solves a problem you actually have right now.

- **The One Question Filter.** When something new drops: "Does this solve a specific pain point I'm currently dealing with?" If yes, test it in a real scenario. If no, save the link, mentally file it, keep walking.

- **North Star Thinking.** Your north star is different from a content creator's. If your north star is building a business, every shiny new tool might be a distraction. The #1 mistake: trying to learn everything, watching every video, testing every tool, jumping to the next thing before the last one had a chance to work.

### The 20% Productivity Dip Rule
When switching tools, expect a ~20% productivity dip in the first two weeks. The question isn't "will this be hard" but "is the dip worth the gain on the other side?" Use this to quantify whether switching tools will ultimately boost output.

### Harnesses Are Interchangeable
"Harnesses are all different on the surface, but they all work inside the same directory." The project directories and workflows should outlive any single tool. Think about what will never change, not what will change.

### Productivity Redefined
Productivity is not how many hours you worked. It's how many meaningful outputs you created that actually moved the needle toward your north star. Someone working 12 hours watching tutorials makes less progress than someone working 5 hours shipping the one thing that matters.

## Positions and Opinions

- Claude Code is THE daily driver. Everything else is supplementary.
- Tool fatigue is real and most people are drowning in it. The cure is a decision framework, not a bigger stack.
- Build working directories that survive tool shifts. Tools are disposable, project structure is not.
- Stop measuring productivity in hours. Measure it in needle-moved-per-hour.
- If you've been contributing to someone's overwhelm with daily content uploads, that's worth acknowledging.
- Most people quit during the 20% dip. Don't.

## How This Applies as a Decision Lens

When evaluating whether to adopt a new tool, framework, or approach, apply the one-question filter: does this solve a specific pain point right now? This is especially relevant for BGD's pivot -- don't chase every new AI tool or framework. Pick the stack that works, build project structures that outlive individual tools, and measure success by output, not by how many tools you've tried. Also validates Justin's choice of Claude Code as the primary operating surface.
