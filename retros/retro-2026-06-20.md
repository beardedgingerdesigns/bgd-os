## Ranked patterns

### 1. Session re-orientation + manual handoff retrieval — the universal tax  ⭐ #1 candidate

- **Type:** NAVIGATION/ROUTER FAILURE · **Fix type:** skill + hook + config
- **Cross-project:** appears in **all 5 corpora** — the only pattern that does.
- **Weekly cost:** HIGH
- **Evidence:**
  - brandos [06-13 04:42] "where did we leave off? /tmp/HANDOFF-brandos-20260613.md" · [06-16 08:27] "pickup where we left off /tmp/brandos-phase10-handoff-2026-06-16.md" (14 handoff refs, 10 distinct `/tmp/*handoff*.md` files)
  - tonequest [06-20 10:10] "so tell me where we are at?" · [06-19 19:57] "pickup where we left off /tmp/tonequest-handoff-2026-06-19.md"
  - small-repos [06-13 14:14] "is there a handoff fiel for mr gym? or where did we last leave off?" · thermal-kitchen [06-12 06:12] "pickup where we left off /tmp/thermal-kitchen-handoff-2026-06-12.md"
  - claude-os [06-18 20:24] "the afternoon schedule should have run. where did that land?"
- **Assessment:** Every session start pays a re-orientation tax. Handoffs are written to ad-hoc `/tmp/*-handoff-DATE.md` paths Justin then has to remember and paste. Three separate agents independently proposed the same fix.
- **Proposed fix:** A user-level **`/resume`** skill that, on session start in any repo, auto-locates the latest handoff (standardize to a stable in-repo path like `docs/handoff-latest.md`, not `/tmp`) plus the repo's STATE, and self-orients — Justin never types "where did we leave off" or pastes a path again.

### 2. CE/GSD/wrap pipeline recited by hand every session

- **Type:** REPETITION · **Fix type:** skill
- **Cross-project:** brandos + claude-os (both big corpora). **Both independently named the fix `/ship`.**
- **Weekly cost:** HIGH
- **Evidence:**
  - brandos [06-18 14:06] "run all of this through /ce-plan and then run /ce-work and then /ce-compound" · [06-19 00:25] "Run it end to end via sub agents from plan to work to wrap"
  - claude-os [06-19 05:45] "run /ce-work and then /ce-compound and then /wrap" · [06-16 10:08] "can we fold /ce-compound into /wrap? one command to finish everything?"
- **Assessment:** Justin recites the same plan→work→compound→wrap chain 25+ times across the week. The chain is deterministic; reciting it is pure overhead.
- **Proposed fix:** One **`/ship`** orchestrator skill running brainstorm→plan→work→simplify→compound→wrap end-to-end via subagents, skipping code-only stages for non-code sessions.

### 3. Orphaned background/cowork agents flood the session

- **Type:** FRICTION/CHURN · **Fix type:** hook + rule
- **Cross-project:** claude-os + small-repos/home.
- **Weekly cost:** HIGH
- **Evidence:**
  - claude-os: 22 idle-notification interrupts (9 on 6/18, 13 on 6/19) + [06-13 04:13] "can you stop them?" · [06-13 04:08] "stop. i believe this is running already via dispatch"
  - home [06-19 05:52] "is claude still running?" → [05:55] "what happened?" (a background screenshot task had died silently at 05:54)
- **Assessment:** The global CLAUDE.md already *describes* the no-background-agents-for-short-tasks rule; it isn't *enforced*. The cost is real and recurring.
- **Proposed fix:** A hook that keeps sub-60s tasks foreground and auto-abandons a background agent after 2 idle pings with no result.

### 4. "What actually got accomplished? I don't see it in the codebase" — deliverable verification gap

- **Type:** FRICTION/CHURN · **Fix type:** rule
- **Weekly cost:** HIGH (tonequest), corroborated by the handoff pattern elsewhere
- **Evidence:**
  - tonequest [06-20 10:13] "I am confused on what was accomplished. i don't see anything in the codebase." · [06-19 17:46] "ok so what do you need me to review?"
- **Assessment:** After background/scratch-DB runs, success gets reported without a concrete artifact. Justin can't tell what persisted vs. what ran in a sandbox.
- **Proposed fix:** Rule — never report success without naming the persisted artifact path, or stating explicitly "this ran in scratch; nothing was written to the repo." (Pairs with #3.)

### 5. Screenshot-as-bug-report — manual UI-feedback relay

- **Type:** MANUAL PROCESS · **Fix type:** skill
- **Weekly cost:** HIGH (brandos: 20 messages); also observed live in today's session (3×)
- **Evidence:**
  - brandos [06-15 11:00] "the modal UI needs quite a bit of help. '/Users/.../Desktop/Screenshot 2026-06-15 at 6.00.23 AM.png'" · [06-19 13:20] "the launch panel takes up too much of the screen... '/Users/.../Screenshot 2026-06-19 at 1.20.52 PM.png'"
- **Assessment:** Justin pastes a full Desktop screenshot path as the feedback payload constantly. Typing/managing the path is friction around an otherwise one-line critique.
- **Proposed fix:** A **`/ss`** skill that auto-grabs the newest `~/Desktop/Screenshot*.png` so Justin types only the critique.
